# Conflict Resolution

Conflitos de merge ocorrem quando duas branches alteram a mesma parte de um arquivo.

Passos para resolver:
1. identifique os arquivos em conflito
2. abra as marcas `<<<<<<<`, `=======`, `>>>>>>>`
3. escolha ou combine as alterações
4. salve o arquivo
5. execute `git add arquivo`
6. finalize com `git commit`

Resolver conflitos com cuidado evita bugs e perdas de trabalho.
