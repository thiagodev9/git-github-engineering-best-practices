# Naming Conventions

Nomes consistentes tornam o código mais fácil de ler.

Recomendações:
- use `feature/` para branches de funcionalidades
- use `bugfix/` ou `hotfix/` para correções
- nomeie arquivos com termos descritivos
- mantenha padrão entre a equipe

Boas convenções reduzem dúvidas e aceleram o desenvolvimento.
