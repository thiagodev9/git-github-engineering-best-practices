# Gitflow

O Gitflow é uma estratégia de branching que define ramos claros para desenvolvimento, releases e correções.

Principais branches:
- `main`: código pronto para produção.
- `develop`: branch de integração contínua.
- `feature/*`: novas funcionalidades.
- `release/*`: preparação para lançamento.
- `hotfix/*`: correções emergenciais em produção.

Use Gitflow quando você precisar de um processo mais rígido e organizado para equipes maiores.
